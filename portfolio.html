TROUBLE SHOOTING JWT 토큰 자동 갱신 — Dio Interceptor SmithLife · Flutter
PROBLEM
문제 현상Access Token 만료 시 모든 API 요청이 401로 실패하며 사용자가 강제 로그아웃됨
원인토큰 만료를 앱 레벨에서 처리하는 로직 없이 단순 요청만 수행. 401 응답을 에러로만 처리하여 재시도 없이 로그인 화면으로 이동
추가 확인여러 요청이 동시에 만료되면 Refresh 요청도 중복 발생하는 race condition 존재
BEFORE / AFTER
항목BeforeAfter
토큰 만료 시강제 로그아웃자동 갱신 후 정상 응답
사용자 경험재로그인 필요끊김 없이 사용 가능
중복 Refreshrace condition 발생단일 갱신 후 대기 처리
RESULT
✅ 자동 갱신 성공
401 감지 → Refresh → 원래 요청 재시도까지 사용자에게 완전히 투명하게 처리
✅ Race Condition 해결
갱신 중 들어오는 요청은 새 토큰 발급 완료 후 일괄 재시도
CODE — auth_interceptor.dart
// 401 응답 감지 → 토큰 갱신 → 재시도 @override void onError(DioException err, handler) async { if (err.response?.statusCode == 401) { final refresh = await TokenStorage.getRefreshToken(); if (refresh == null) { return handler.next(err); } try { // Refresh Token으로 새 Access Token 발급 final res = await _dio.post( ApiConstants.refresh, data: {'refreshToken': refresh}, ); final newToken = res.data['data'] ['accessToken']; // 저장 후 원래 요청 헤더 교체 await TokenStorage.saveAccessToken(newToken); err.requestOptions.headers['Authorization'] = 'Bearer $newToken'; // 원래 요청 재시도 final retried = await _dio.fetch( err.requestOptions ); return handler.resolve(retried); } catch (_) { // Refresh 실패 → 로그아웃 await TokenStorage.clearTokens(); handler.next(err); } } else { handler.next(err); } }
